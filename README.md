# 小红书文案管理系统（纯前端版）

一个用于编辑和展示小红书文案的完整系统，无需安装任何依赖，直接浏览器打开即可使用。

## 功能特点

- **纯前端运行**: 无需Node.js服务器，无需安装npm依赖
- **本地存储**: 文案数据保存在浏览器本地存储中
- **编辑页面** (`admin.html`): 可视化编辑文案，实时预览
- **展示页面** (`index.html`): 显示文案并支持一键复制
- **跨页面同步**: 编辑后立即生效，无需刷新

## 文件结构

```
├── index.html          # 展示页面（用户端）
├── admin.html         # 管理页面（编辑端）
└── README.md          # 使用说明
```

## 使用方法

### 1. 直接打开文件

- **展示页面**: 双击打开 `index.html`
- **管理页面**: 双击打开 `admin.html`

### 2. 使用本地服务器（推荐）

如果你想要更好的开发体验，可以使用简单的Python HTTP服务器：

```bash
# Python 3
python -m http.server 8000

# 或 Python 2
python -m SimpleHTTPServer 8000
```

然后访问：
- http://localhost:8000/index.html
- http://localhost:8000/admin.html

## 使用说明

### 编辑文案

1. 打开管理页面 (`admin.html`)
2. 编辑标题和文案内容
3. 点击"保存文案"按钮
4. 文案会自动保存到浏览器的本地存储中

### 查看和复制文案

1. 打开展示页面 (`index.html`)
2. 页面自动加载最新保存的文案
3. 点击"复制标题"或"复制文案"按钮
4. 直接粘贴到小红书

## 技术特点

- **纯HTML/CSS/JavaScript**: 无任何外部依赖
- **本地存储**: 使用 `localStorage` API保存数据
- **响应式设计**: 完美适配桌面和移动设备
- **实时预览**: 编辑时即时预览效果

## 数据存储

文案数据保存在浏览器的本地存储中，键名为 `xiaohongshu_texts`。这意味着：

- 数据保存在当前浏览器中
- 清空浏览器缓存会删除文案数据
- 不同浏览器的数据不共享
- 数据不会上传到任何服务器

## 浏览器兼容性

支持所有现代浏览器：
- Chrome 4+
- Firefox 3.5+
- Safari 4+
- Opera 10.5+
- Edge 12+
- iOS Safari 3.2+

## 注意事项

1. **数据安全**: 文案只保存在本地浏览器，不会上传到网络
2. **浏览器限制**: 不同浏览器/设备的本地存储不共享
3. **缓存清理**: 清空浏览器缓存会删除保存的文案
4. **备份建议**: 定期复制重要文案到其他地方备份
