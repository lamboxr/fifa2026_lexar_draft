在路由器上不使用任何后端语言（PHP/Python/Node.js 等），仅通过 Nginx 本身的功能，完全可以实现“一个网页编辑保存文本，另一个网页动态读取”的需求。核心思路是 **利用 Nginx 的 WebDAV 模块**，让浏览器通过 HTTP 的 `PUT` 方法直接写入文件，通过 `GET` 读取文件。

---

### 实现原理

- **保存**：前端使用 `fetch` 发起 `PUT` 请求，将文本内容作为请求体发送到 Nginx 的某个路径（如 `/data/mydata.txt`）。
- **读取**：前端使用 `fetch` 发起 `GET` 请求，读取同一个路径的内容并显示。
- Nginx 负责处理 `PUT` 请求，将内容保存为静态文件，并响应 `GET` 请求提供文件内容。

---

### 第一步：配置 Nginx 支持 WebDAV

编辑 `/etc/nginx/uci.conf`，在 `http` 块内添加一个专门用于数据存储的 `location`，启用 WebDAV 方法：

```nginx
http {
    # ... 其他配置不变 ...

    server {
        listen 8880;
        listen [::]:8880;
        server_name _;
        root /www;                     # 默认根，但 location 会覆盖

        # 新增一个 location 用于存储和读取数据
        location /data/ {
            alias /mnt/mmc0-4/webdata/;   # 实际存储路径，确保可写
            dav_methods PUT DELETE MKCOL COPY MOVE;
            dav_access user:rw group:rw all:r;
            autoindex on;                 # 可选，列出文件列表
            allow all;
        }

        # 原有的 location 配置（如静态网站）可以保留
    }
}
```

**重要说明**：
- `alias` 指向一个真实存在的目录（如 `/mnt/mmc0-4/webdata/`），确保 Nginx 进程有**写入权限**。
- 如果目录不存在，先创建并授权：
  ```bash
  mkdir -p /mnt/mmc0-4/webdata/
  chown -R network:network /mnt/mmc0-4/webdata/   # 假设 Nginx 用户是 network
  chmod -R 755 /mnt/mmc0-4/webdata/
  ```
  可通过 `ps | grep nginx | grep worker` 查看 Nginx 运行用户。

重启 Nginx：
```bash
killall nginx
nginx -c /etc/nginx/uci.conf
```

### 第二步：前端页面（编辑保存）

创建一个 HTML 页面（例如 `editor.html`），放置于 `/www/` 下，或任意可访问位置：

```html
<!DOCTYPE html>
<html>
<head>
    <title>文本编辑器</title>
    <style>textarea { width: 100%; height: 300px; }</style>
</head>
<body>
    <h1>编辑文本</h1>
    <textarea id="content">在这里输入内容...</textarea><br>
    <button onclick="save()">保存</button>
    <div id="status"></div>

    <script>
        const FILE_URL = '/data/mydata.txt';   // 对应 Nginx 的 location

        function save() {
            const content = document.getElementById('content').value;
            fetch(FILE_URL, {
                method: 'PUT',
                body: content,
                headers: { 'Content-Type': 'text/plain' }
            })
            .then(res => {
                if (res.ok) {
                    document.getElementById('status').innerText = '保存成功！';
                } else {
                    document.getElementById('status').innerText = '保存失败：' + res.status;
                }
            })
            .catch(err => {
                document.getElementById('status').innerText = '错误：' + err;
            });
        }

        // 加载已有内容（可选）
        fetch(FILE_URL)
            .then(res => {
                if (res.ok) return res.text();
                throw new Error('文件不存在');
            })
            .then(text => {
                document.getElementById('content').value = text;
            })
            .catch(() => {}); // 忽略错误，保持空白
    </script>
</body>
</html>
```

### 第三步：前端页面（动态读取）

创建另一个页面（例如 `viewer.html`），用于实时显示数据：

```html
<!DOCTYPE html>
<html>
<head>
    <title>查看文本</title>
    <style>pre { white-space: pre-wrap; }</style>
</head>
<body>
    <h1>实时文本内容</h1>
    <pre id="display"></pre>
    <button onclick="load()">刷新</button>

    <script>
        const FILE_URL = '/data/mydata.txt';

        function load() {
            fetch(FILE_URL)
                .then(res => {
                    if (!res.ok) throw new Error('读取失败');
                    return res.text();
                })
                .then(text => {
                    document.getElementById('display').textContent = text;
                })
                .catch(err => {
                    document.getElementById('display').textContent = '读取错误：' + err;
                });
        }

        // 自动刷新（每5秒）
        setInterval(load, 5000);
        load();
    </script>
</body>
</html>
```

---

### 访问与测试

- 将两个 HTML 文件放到 Nginx 可服务的目录（如 `/www/`）。
- 内网访问：
  - `http://路由器IP:8880/editor.html` → 编辑保存
  - `http://路由器IP:8880/viewer.html` → 动态查看

保存的数据会以文件形式存储在 `/mnt/mmc0-4/webdata/mydata.txt`，即使重启路由器也不会丢失。

---

### 注意事项

1. **安全性**：此方案允许任何能访问该端口的人读写文件，适合内网环境。若要限定 IP，可配合 `allow/deny` 指令。
2. **跨域**：如果前后端端口不同，需配置 CORS，但这里同端口同域，无跨域问题。
3. **文件覆盖**：`PUT` 会覆盖现有文件，如需追加可自行设计。
4. **权限问题**：务必确保 Nginx 进程对存储目录有写权限，否则 `PUT` 会返回 `403` 或 `500`。
5. **大文件**：`client_max_body_size` 可调整上传限制。

---

### 扩展：自动刷新与事件驱动

如果你希望“另一个网页动态读取”无需手动刷新，可以使用 `EventSource` 或 WebSocket，但需要后端支持。更简单的是前端定时轮询（如上述 `setInterval`），对文本数据足够用。

---

### 总结

利用 Nginx 的 WebDAV 模块，无需任何后端脚本，纯静态 + HTTP 方法即可实现文本的保存与读取。此方法轻量、稳定，非常适合 OpenWrt 这类嵌入式系统。若需要更复杂的功能（如用户认证、多文件管理），可以进一步扩展 Nginx 配置。
